# Day15 요약 및 헷갈리는 개념

## 예외처리

* 예외처리 정의 : 프로그램 실행 중, 예외 발생에 대비한 코드 작성.
* 예외처리 목적 : 프로그램의 비정상적 종료를 막기 위함.

**예외처리**

1. 직접처리
2. 보고
3. 은폐
4. 예외되던지기 (직접처리, 보고 포함)

* 컴파일 에러 : 컴파일 과정에서 발생
* 런타임 에러 : 실행중(런타임) 과정에서 발생 (비정상 정료)
    * 예외 : 덜심각 - 처리가능 (예외처리)
        * Exception : 사용자 잘못 - 필수처리
        * RuntimeException : 프로그래머 잘못 - 선택처리
    * 에러 : 심각 - 복구불가
* 논리적 에러 : 실행중 발생하며 프로그램의 의도와 다르게 동작하는 경우
